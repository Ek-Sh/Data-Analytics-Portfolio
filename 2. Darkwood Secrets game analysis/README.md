# Анализ поведения игроков онлайн-игры «Секреты Темнолесья»


***СТАТУС:*** **Завершён**


## Цель проекта:

На основе данных онлайн-игры «Секреты Темнолесья» изучить поведение игроков и проверить:

1. Какая доля игроков покупает премиальную валюту «райские лепестки».
2. Зависит ли доля платящих игроков от расы персонажа.
3. Какие эпические предметы покупают чаще всего.
4. Есть ли различия в покупательской активности игроков разных рас.


## Описание проекта:

В проекте анализируются данные онлайн-игры, где пользователь выбирает расу, класс персонажа и легендарную способность. Эти характеристики могут влиять на прохождение игры и покупательское поведение.

Премиальная валюта «райские лепестки» используется для покупки эпических предметов и является важной частью монетизации игры.

С помощью SQL-запросов рассчитаны доля платящих игроков, статистика покупок, популярность предметов и активность игроков в разрезе рас.


## Вывод исследования:

1. Платящие игроки составляют небольшую часть от всех зарегистрированных пользователей.
2. Доля платящих игроков различается по расам, но в целом значения близки к среднему уровню по игре.
3. Средняя стоимость покупки выше медианной, что указывает на наличие крупных покупок.
4. В данных есть покупки с нулевой стоимостью, поэтому они исключены из анализа активности.
5. Некоторые эпические предметы покупают чаще остальных, а часть предметов почти не востребована.
6. Покупательская активность отличается между расами по доле покупателей, количеству и стоимости покупок.


## Использование результатов исследования:

Результаты могут быть использованы маркетинговой командой для настройки рекламных кампаний и продвижения эпических предметов.

Исследование помогает определить перспективные сегменты игроков, популярные предметы и возможные признаки дисбаланса между расами.


## Технические особенности проекта:

Данные представлены в виде реляционной базы с таблицами об игроках, покупках, расах, классах, странах, умениях и предметах.

В ходе проекта выполнены:

- проверка корректности поля `payer`;
- анализ покупок с нулевой стоимостью;
- исключение нулевых покупок из основных расчётов;
- поиск редко покупаемых и непокупаемых предметов;
- расчёт показателей активности игроков по расам.

Для анализа использовались SQL-запросы с агрегацией, `JOIN` и CTE.
