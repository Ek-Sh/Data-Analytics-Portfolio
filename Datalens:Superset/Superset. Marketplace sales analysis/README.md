## Цель проекта:

На основе данных маркетплейса «МаркетПро» подготовить аналитическую основу для BI-дашборда, который позволит отслеживать ключевые показатели продаж, пользовательской активности и качества товаров.
В рамках проекта необходимо:
1. Подготовить и предобработать данные маркетплейса.
2. Рассчитать ключевые бизнес-метрики: конверсию, средний чек, количество позиций в заказе и Retention Rate.
3. Проанализировать продажи по времени, категориям и товарам.
4. Выявить категории с наибольшим количеством отменённых заказов.
5. Определить топовых покупателей по сумме заказов.
6. Подготовить данные и рекомендации для построения BI-дашборда.

## Описание проекта:

Компания «МаркетПро» — крупная онлайн-платформа для продаж, где продавцы размещают товары, а пользователи совершают покупки. Отделам маркетинга и продаж необходим удобный дашборд, который поможет быстро отслеживать ключевые показатели бизнеса и принимать решения на основе данных.
В проекте используются данные о пользователях, товарах, категориях, заказах, позициях заказов, отзывах и транзакциях. На этапе предобработки данные были проверены на пропуски, дубликаты, некорректные типы, аномалии и логические ошибки. Дополнительно была подготовлена выгрузка заказов за период с 01 января по 01 июня 2025 года.
После подготовки данных были рассчитаны основные метрики маркетплейса: сумма продаж, количество заказов, средний чек, конверсия пользователей в покупателей, среднее количество позиций в заказе и Retention Rate по когортам. Также была выполнена ad-hoc аналитика по срочным запросам заказчика: анализ отменённых заказов по категориям, определение топ-10 покупателей и расчёт динамики выручки по пользователям.

## Вывод исследования:

1. Подготовлена структура данных для дальнейшего построения BI-дашборда продаж и пользовательской активности маркетплейса.
2. Данные были проверены и очищены: обработаны пропуски, дубликаты, некорректные значения и логические ошибки.
3. Рассчитаны ключевые бизнес-метрики, которые позволяют оценивать эффективность маркетплейса: конверсия, средний чек, количество позиций в заказе и Retention Rate.
4. Определены направления для анализа качества работы платформы: категории с отменёнными заказами, товары с отзывами и рейтингами, активность покупателей и продавцов.
5. Подготовлены SQL-запросы для решения ad-hoc задач бизнеса: анализа отмен, поиска наиболее ценных покупателей и оценки динамики выручки по месяцам.
6. Сформированы рекомендации по структуре будущего дашборда, включая основные KPI, фильтры и визуализации.

## Технические особенности проекта:
В проекте использовались данные из нескольких связанных таблиц:
* categories — категории товаров;
* products — товары маркетплейса;
* orders — заказы пользователей;
* order_items — позиции товаров в заказах;
* users — пользователи платформы;
* reviews — отзывы и оценки товаров;
* transactions — платежи и возвраты.

На этапе предобработки были выполнены:
1. Загрузка данных из CSV-файлов.
2. Проверка структуры и типов данных.
3. Поиск и обработка пропусков.
4. Удаление явных и неявных дубликатов.
5. Проверка логических ошибок и аномальных значений.
6. Преобразование дат к корректному формату.

Для расчёта бизнес-метрик использовались SQL-запросы.
