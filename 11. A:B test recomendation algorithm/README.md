# A/B-тестирование алгоритма рекомендаций в развлекательном приложении

***СТАТУС:*** **Завершён**

##Цель проекта:

Рассчитать параметры A/B-теста, проверить корректность его проведения и проанализировать результаты эксперимента для развлекательного приложения с бесконечной лентой контента.
В рамках проекта проверялась бизнес-гипотеза: новый алгоритм рекомендаций показывает пользователям более интересный контент, поэтому должен повышать вовлечённость в приложении. В качестве основной метрики была выбрана доля «успешных» сессий, в которых пользователь просмотрел 4 и более страницы.  

##Описание проекта:

Компания разрабатывает развлекательное приложение с бесконечной лентой, похожей на приложения с короткими видео. В продукте используются две модели монетизации: платная ежемесячная подписка без рекламы и показ рекламы пользователям, которые не оформили подписку.
Команда рекомендательных систем разработала новый алгоритм персональных рекомендаций. Перед внедрением алгоритма необходимо было оценить, действительно ли он повышает пользовательскую вовлечённость.

Для анализа использовались три набора данных:

1. sessions_project_history — исторические данные по сессиям пользователей за период с 2025-08-11 по 2025-09-23.
2. sessions_project_test_part — данные за первый день A/B-теста, 2025-10-14.
3. sessions_project_test — данные за весь период A/B-теста с 2025-10-14 по 2025-11-02.

В ходе проекта были выполнены EDA, расчёт параметров эксперимента, проверка корректности разбиения пользователей по группам и статистический анализ результатов теста.

##Вывод исследования:
1. На исторических данных была выбрана основная метрика эксперимента — доля успешных сессий, где пользователь просмотрел не менее 4 страниц. Такая метрика отражает вовлечённость пользователя в ленту и позволяет оценить качество рекомендаций.
2. Базовый уровень доли успешных сессий по историческим данным держался примерно на уровне 30%. Это позволило использовать исторические данные для планирования эксперимента и расчёта необходимого размера выборки.
3. Для A/B-теста были заданы параметры: уровень значимости 0.05, мощность теста 80%, минимальный детектируемый эффект около 3% относительно базового уровня. Необходимый размер выборки составил около 82 081 наблюдения на каждую группу.
4. На первом этапе мониторинга теста было проверено распределение пользователей между группами. В группе A было 1477 пользователей, в группе B — 1466 пользователей. Разница составила 0,74%, что не является существенным перекосом.
5. Пересечений пользователей между группами A и B не обнаружено. Это означает, что выборки независимы, а пользователи не попали одновременно в контрольную и тестовую группы.
6. Распределение пользователей по устройствам и регионам в группах A и B оказалось сопоставимым. Явных перекосов в структуре аудитории не выявлено.
7. По итогам всего эксперимента среднее число сессий в день в группе A составило 2477,55, в группе B — 2522,7. T-тест показал p-value ≈ 0.938, поэтому статистически значимых различий по объёму трафика между группами нет.
8. Доля успешных сессий в контрольной группе A составила 30,77%, а в тестовой группе B — 31,83%. Абсолютный прирост составил около 1,1 процентного пункта.
9. Для проверки статистической значимости использовался z-тест для двух пропорций. Значение p-value составило 0.0003, что ниже уровня значимости 0.05. Следовательно, различие между группами статистически значимо.
10. Новый алгоритм рекомендаций положительно повлиял на вовлечённость пользователей: в тестовой группе пользователи чаще просматривали 4 и более страницы за сессию.

##Использование результатов исследования:

Результаты исследования могут быть использованы продуктовой командой и командой рекомендательных систем для принятия решения о внедрении нового алгоритма рекомендаций.
Так как эксперимент показал статистически значимый рост ключевой метрики, новый алгоритм можно рекомендовать к внедрению на всю аудиторию приложения. При этом после раскатки важно продолжить мониторинг смежных бизнес-метрик: конверсии в регистрацию, конверсии в подписку, рекламной выручки, retention и длительности сессии.

##Технические особенности проекта:

В проекте была проведена работа с историческими и экспериментальными данными по сессиям пользователей. Для подготовки A/B-теста были рассчитаны базовый уровень метрики, минимальный детектируемый эффект, необходимый размер выборки и предполагаемая длительность теста.

Для проверки корректности эксперимента анализировались:
* распределение пользователей по группам;
* отсутствие пересечений пользователей между группами;
* равномерность распределения по устройствам;
* равномерность распределения по регионам;
* сопоставимость групп по количеству дневных сессий.

Для анализа результатов эксперимента была создана бинарная метрика good_session, принимающая значение 1, если пользователь просмотрел 4 и более страницы за сессию, и 0 — если меньше.
Статистическая проверка проводилась с помощью z-теста для двух долей. Дополнительно использовался t-тест для оценки сопоставимости среднего количества сессий в день между группами.
